# FDE养成计划使用与配置说明

这份说明给安装 `fde-growth-plan` 的用户使用。它解释如何开始每日学习、如何设置个性化信息，以及每日内容为什么需要平台自动任务来触发。

## 1. 这个skill能做什么

`fde-growth-plan` 是一个 FDE（Forward Deployed Engineer，前沿部署工程师）每日学习教练。它会围绕 AI 应用工程、部署交付、推理优化、可观测性、客户沟通、产品反馈闭环和面试准备，生成每日学习内容和动手任务。

它适合三类人：

1. 刚了解 FDE，想系统判断这个方向是否适合自己。
2. 已经做 AI 产品、工程、交付、售前、咨询或解决方案，想向 FDE 转型。
3. 想准备 FDE/AI Solutions Engineer/Forward Deployed AI Engineer 类岗位面试和作品集。

## 2. 推送方式说明

Skill 本身不会凭空定时发消息。它负责“生成什么、按什么节奏学、怎么沉淀”，但“每天几点触发”需要由飞行社、WorkBuddy、日历提醒、机器人或其他自动任务能力来完成。

你可以用三种方式：

### 方式A：手动触发

每天对助手说：

```text
今天的FDE学习
```

适合刚试用，或不想每天被自动打扰的人。

### 方式B：平台自动任务

如果飞行社/WorkBuddy 支持每日自动任务，建议设置：

- 名称：FDE每日学习
- 时间：每天早上 8:00，或你固定学习的时间
- 提示词：

```text
请使用 fde-growth-plan skill，基于我的用户配置和学习进度，生成今天的FDE每日学习内容。内容要包含核心概念、真实案例、今日任务、验收标准、进阶思考和来源链接；同时更新学习笔记。
```

### 方式C：外部提醒 + 手动触发

如果平台暂时不支持自动推送，可以先用日历、飞书提醒或手机提醒每天提醒自己，然后回到助手里发送“今天的FDE学习”。

## 3. 首次配置

第一次使用时，建议直接发送：

```text
配置我的FDE养成计划
```

也可以直接发送：

```text
今天的FDE学习
```

如果这是第一次使用，助手会先进入 Day 1 摸底，而不是直接开始讲课。摸底的目的不是考试，而是让后续每日学习内容更合适：该讲浅一点还是深一点，该多做代码还是多做客户场景，该用4周体验还是24周系统版。

## 5. 个性化字段怎么影响每日内容

### 当前背景

决定解释方式和起点。

- 产品/增长背景：更多解释工程概念，强化场景、ROI、客户沟通和产品反馈闭环。
- 软件工程背景：减少基础编程解释，增加部署、监控、架构和代码任务。
- 算法/研究背景：强化生产工程、API、云部署和客户交付。
- 售前/咨询背景：强化技术实现、Demo到生产、评估指标和代码作品集。

### 每日投入时间

决定任务大小。

- 15分钟：一张知识卡 + 一个思考题。
- 30分钟：一个概念 + 一个小任务。
- 60分钟：概念 + 案例 + 可运行练习。
- 90分钟以上：项目推进、代码实现、部署或面试模拟。

### 目标

决定路线偏向。

- 了解方向：先讲岗位、案例、能力地图、真实工作流。
- 求职转型：更早加入作品集、面试题和项目包装。
- 工作提升：围绕真实业务场景设计任务。
- 创业/产品探索：强化客户问题、ROI、产品化抽象和交付闭环。

### 学习周期

决定主题取舍和推进速度。

- 4周体验版：适合先判断自己是否喜欢FDE。重点是岗位理解、AI应用基础、一个轻量案例和一次复盘。
- 8周入门版：适合建立基本能力框架。重点是Prompt/RAG/Agent/Evals，加上最小部署和一个可演示作品。
- 12周转型版：适合准备求职或内部转岗。重点是AI应用、部署监控、客户需求拆解、作品集包装和面试表达。
- 24周系统版：适合认真转型。完整覆盖五横三纵、生产交付、私有化部署、客户沟通、反馈闭环和面试冲刺。
- 自定义周期：适合有明确截止日期的人。助手会按剩余周数压缩主题，保留最关键能力，删掉低优先级扩展内容。

如果你不确定选哪个，建议先选 4周体验版。体验结束后再升级到 12周或24周。

### 偏好行业

决定案例和项目选题。

- 金融：合规、风控、审计、私有化部署、数据权限。
- 医疗：隐私、准确性、人工复核、临床工作流。
- 制造：边缘部署、设备数据、异常检测、现场集成。
- 企业知识库：RAG、权限、检索质量、知识治理。
- 客服/销售：Agent、质检、工具调用、人工接管。

## 6. 每日内容格式

每天的输出建议长这样：

```markdown
# Day X：今日主题

## 你今天要理解什么
用通俗语言解释核心概念。

## 为什么FDE必须懂它
连接客户现场、生产部署、业务价值或产品反馈。

## 真实案例/行业信号
列出官方资料、岗位JD、公司博客或可信报道，并说明可信度。

## 今日任务
按你的时间预算给出可执行任务。

## 验收标准
明确今天做完的标志。

## 进阶思考
1-3个问题，训练系统思维和客户视角。

## 来源
列出链接；没有联网核验时必须明确说明。
```

## 7. 学习笔记

默认学习笔记保存在 skill 目录下：

```text
learning-notes.md
```

如果你希望保存到别处，可以在配置里写：

```text
学习笔记保存位置：例如 ~/Documents/FDE养成学习笔记.md
```

每日学习后，助手应把当天内容追加到学习笔记顶部，方便倒序复盘。

## 8. 周复盘怎么用

每周末发送：

```text
复盘本周FDE学习
```

助手会总结：

1. 本周学了什么。
2. 哪些任务完成了。
3. 五横三纵里训练了哪些能力。
4. 下周应该补什么。
5. 是否能沉淀成作品集材料。

## 9. 常见问题

### 为什么有时要联网？

FDE 是新兴岗位，岗位JD、公司组织方式、AI工具、模型、法规和部署案例变化很快。涉及“最新”“今天”“某公司岗位要求”“薪资”“法规”“工具版本”时，必须联网核验。

### 没有工程基础能用吗？

能用，但要把每日任务设置为“轻量阅读 + 小练习”。不要一开始就要求自己做K8s、推理优化或完整项目。

### 这个skill能保证我6个月成为FDE吗？

不能保证。它能提供路线、任务、复盘和作品集训练，但 FDE 的核心门槛是真实交付经验。建议从第2个月开始，尽量找真实业务问题做小项目。

### 学习内容会不会太硬核？

取决于配置。请明确写出每日可投入时间、工程基础和希望难度。配置越清楚，内容越贴合。
