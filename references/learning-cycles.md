# FDE养成计划学习周期

使用本文件把默认 24 周路线压缩或展开。用户配置了学习周期时，优先按本文件选择主题；没有配置时，先在 Day 1 做摸底，再建议周期。用户只是试用或暂不确定时，默认 4 周体验版。

## Day 1：统一摸底日

无论用户选择 4周、8周、12周、24周还是自定义周期，第一天都先做摸底。Day 1 的目标是建立个人学习配置，而不是直接进入技术细节。

Day 1 必须完成：

1. 解释 FDE 是什么，以及它和售前、交付、咨询、软件工程的区别。
2. 收集用户背景、学习目标、学习周期、每日学习时长、技术基础、行业偏好和输出偏好。
3. 完成 8 项能力自评：AI应用理解、Python/API、RAG/Agent/Evals、部署/云服务、监控/质量评估、客户沟通、业务ROI表达、项目复盘沉淀。
4. 根据摸底结果给出起点画像和学习路线建议。
5. 确认 Day 2 的主题。

如果用户已经提前填写了配置，Day 1 应改为“配置解读 + FDE岗位全景 + 能力差距分析”。

## 4周体验版

目标：判断自己是否适合 FDE，建立岗位认知，完成一个轻量输出。

| 周 | 主题 | 重点 | 交付物 |
|---|---|---|---|
| 第1周 | FDE岗位理解 | FDE vs 售前/交付/咨询/软件工程，五横三纵，真实工作流 | 个人能力差距表 |
| 第2周 | LLM应用基础 | Prompt、RAG、Agent、Evals的最小概念 | 一个AI应用方案或轻量Demo |
| 第3周 | 客户问题拆解 | 模糊需求、ROI、MVP、客户演示 | 一页客户方案 |
| 第4周 | 体验项目与复盘 | 选一个行业场景，从问题到方案/原型 | 体验总结和下一阶段建议 |

适合：刚了解FDE、想试试看、每天15-30分钟的人。

## 8周入门版

目标：掌握 AI 应用和基础交付链路，做出一个可演示作品。

| 周 | 主题 | 重点 | 交付物 |
|---|---|---|---|
| 第1周 | FDE岗位与能力地图 | 岗位实质、典型JD、能力差距 | 能力差距表 |
| 第2周 | Prompt与工具调用 | Prompt、安全、Function Calling | 小型工具调用Demo |
| 第3周 | RAG系统 | 文档解析、检索、重排序、评估 | RAG问答Demo |
| 第4周 | Agent与Evals | Agent工作流、质量评估、反馈闭环 | Demo评估报告 |
| 第5周 | API与容器化 | FastAPI、Docker、基础部署 | 可运行服务 |
| 第6周 | 监控与成本 | 日志、指标、Token成本、错误率 | 简单监控面板或指标表 |
| 第7周 | 客户场景项目 | 需求拆解、MVP、ROI、方案设计 | 客户方案文档 |
| 第8周 | 作品集包装 | README、架构图、Demo讲解 | GitHub/作品集页面草稿 |

适合：希望快速入门、每天30-60分钟的人。

## 12周转型版

目标：面向求职或内部转岗，建立作品集和面试表达。

| 周 | 主题 | 重点 | 交付物 |
|---|---|---|---|
| 第1周 | FDE岗位理解与目标定位 | JD、能力地图、目标岗位 | 转型计划 |
| 第2周 | Prompt与安全 | Prompt、注入防护、输出校验 | Prompt评估集 |
| 第3周 | RAG | 检索、重排序、权限、质量 | RAG Demo |
| 第4周 | Agent与Evals | 工具调用、评估、回归测试 | Agent Demo + Evals |
| 第5周 | 软件工程化 | API、测试、日志、错误处理 | 生产化改造 |
| 第6周 | Docker与部署 | 容器化、配置、环境管理 | 部署文档 |
| 第7周 | 推理与成本 | 模型选择、延迟、吞吐、成本 | 成本/延迟分析 |
| 第8周 | 可观测性 | Metrics、Logs、Tracing、告警 | 监控方案 |
| 第9周 | 客户需求拆解 | 模糊问题、MVP、ROI | 客户需求分析 |
| 第10周 | 行业场景实战 | 选金融/医疗/制造/知识库等 | 端到端项目 |
| 第11周 | 作品集包装 | README、架构图、演示稿 | 可投递作品集 |
| 第12周 | 面试冲刺 | System Design、Role-play、项目深挖 | 3轮模拟面试复盘 |

适合：有一定基础、希望3个月内形成求职材料的人。

## 24周系统版

目标：完整覆盖 FDE 所需能力。按 `references/learning-topics.md` 执行。

四个阶段：

1. 第1-4周：LLM应用全栈筑基。
2. 第5-10周：部署与推理优化。
3. 第11-18周：客户交付实战。
4. 第19-24周：面试冲刺与作品集。

适合：认真转型、每天60分钟左右、愿意做项目的人。

## 自定义周期规则

如果用户给出自定义结束日期或周数，按下面规则压缩：

1. 少于4周：只做岗位认知、能力差距、一个轻量方案，不承诺工程作品。
2. 4-7周：按4周体验版执行，额外时间补RAG或Agent。
3. 8-11周：按8周入门版执行，额外时间补部署或面试。
4. 12-23周：按12周转型版执行，额外时间补推理优化、私有化部署、行业项目。
5. 24周以上：按24周系统版执行，额外时间做更多真实项目和开源贡献。

压缩时保留优先级：

1. FDE岗位理解和能力差距。
2. Prompt/RAG/Agent/Evals基础。
3. 一个可演示的AI应用或方案。
4. 客户需求拆解和ROI表达。
5. 部署、监控、安全和成本。
6. 作品集和面试训练。
