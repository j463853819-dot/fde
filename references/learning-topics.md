# FDE养成计划 — 24周学习主题规划

## 学习体系概述

本学习体系设计为24周（约6个月），每天一个主题，由浅入深，系统化构建完整的FDE知识框架。整个学习路径分为四个阶段：

注意：第1天固定为摸底日。无论后续采用4周、8周、12周还是24周节奏，都先收集用户背景、学习目标、学习周期、每日时长和能力自评分，再开始正式推进。

### 第一阶段：LLM应用全栈筑基（第1-4周）
- 目标：掌握Prompt工程、RAG、Agent、Evals四件套
- 产出：独立搭建一个带真实数据源的端到端AI应用
- 覆盖维度：生产级软件工程

### 第二阶段：部署与推理优化（第5-10周）
- 目标：掌握模型量化、推理加速、K8s部署、监控告警
- 产出：将一个开源模型从代码部署到生产环境
- 覆盖维度：高性能推理优化、云原生基础设施、可观测性与运维

### 第三阶段：客户交付实战（第11-18周）
- 目标：深入FDE核心工作流，完成客户交付模拟项目
- 产出：选定垂直场景，2周内从业务问题到生产级demo
- 覆盖维度：五横全部 + 业务沟通软技能

### 第四阶段：面试冲刺与作品集（第19-24周）
- 目标：准备FDE特色面试，建立完整作品集
- 产出：端到端GitHub作品集 + 3轮模拟面试
- 覆盖维度：全能力体系 + 面试技巧

## 主题生成规则

### 每日主题选择原则
1. **系统性**: 确保主题之间有逻辑关联
2. **实用性**: 选择对实际工作最有帮助的主题
3. **时效性**: 结合最新的行业动态和技术趋势
4. **渐进性**: 难度逐步增加，符合学习曲线

### 内容生成要求
1. **概念清晰**: 用工程语言解释专业概念，兼顾通俗性
2. **案例丰富**: 至少包含1-2个真实企业案例
3. **实践导向**: 每条内容末尾附一个动手任务
4. **深度思考**: 引导用户进行工程思维训练

### 案例选择标准
1. **代表性**: OpenAI、Anthropic、Palantir、Google等FDE鼻祖和领军企业的实践
2. **时效性**: 优先选择近3个月内的案例
3. **多样性**: 覆盖不同行业、不同规模的部署场景
4. **启发性**: 案例要有明确的经验教训
