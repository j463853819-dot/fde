# FDE每日学习内容模板

## 模板使用说明

本模板用于生成每日FDE学习内容，确保内容格式统一、结构完整、深度足够。每次生成内容时，请按照此模板的结构和指导原则进行创作。

---

## {日期} - {今日主题}

### 🎯 核心概念

**目标**: 深入浅出地讲解一个FDE核心概念，覆盖"五横三纵"能力模型。

**内容要求**:
1. **概念定义**: 用一句话清晰定义概念
2. **重要性说明**: 解释为什么这个概念对FDE很重要
3. **关键要点**: 列出3-5个核心要点
4. **常见误区**: 指出1-2个常见的理解误区

**示例结构**:
```
{概念名称}是指{定义}。在FDE工作中，这个概念特别重要，因为{重要性}。

核心要点包括：
1. {要点1}: {解释}
2. {要点2}: {解释}
3. {要点3}: {解释}

需要注意的误区：
- {误区1}: {正确理解}
- {误区2}: {正确理解}
```

### 📊 最新案例

**目标**: 结合真实AI企业的FDE实践案例，让概念更加具体和生动。

**内容要求**:
1. **案例选择**: 优先选择近3个月内的案例
2. **案例数量**: 1-2个不同角度的案例
3. **案例分析**: 每个案例都要有明确的分析框架
4. **经验总结**: 从案例中提炼可借鉴的经验

**示例结构**:
```
1. **{案例1名称}**
   - 背景: {简要背景}
   - 做法: {具体做法}
   - 效果: {产生的效果}
   - 启示: {可借鉴的经验}
```

### 🔮 趋势洞察

**目标**: 关联最新的行业趋势和动态，让学习与时俱进。

**内容要求**:
1. **趋势识别**: 识别当前最重要的AI部署/基础设施趋势
2. **趋势分析**: 分析趋势背后的原因和影响
3. **应对策略**: 提出应对趋势的建议

**示例结构**:
```
当前AI部署领域的主要趋势包括：

1. **{趋势1}**
   - 表现: {趋势的具体表现}
   - 影响: {对FDE工作的影响}
   - 建议: {如何应对}
```

### 🛠️ 动手实践

**目标**: 提供可立即执行的动手任务，将知识转化为工程能力。

**内容要求**:
1. **操作导向**: 每个任务步骤清晰可执行
2. **资源支持**: 提供必要的命令/代码/工具链参考
3. **验收标准**: 明确任务完成的标准

**示例结构**:
```
今日动手任务：{任务名称}

步骤：
1. {步骤1}
2. {步骤2}
3. {步骤3}

验收标准：
- {标准1}
- {标准2}

📦 参考资源:
- {链接/命令/代码片段}
```

### 💭 进阶思考

**目标**: 引导用户进行深度思考，培养FDE的系统思维。

**示例结构**:
```
深入思考问题：

🤔 批判性思考
1. {问题}: {思考指引}

🔍 联系实际
1. {问题}: 在你的项目中如何体现？

🚀 探索延伸
1. {问题}: 可以进一步研究什么？
```

## 内容生成流程

### 第1步: 主题选择
- 参考 `references/learning-topics.md` 中的主题规划
- 考虑当前学习阶段的系统性
- 结合最新行业动态

### 第2步: 资料收集
- 查阅 `references/case-library.md` 寻找相关案例
- 搜索最新的AI部署新闻和技术更新
- 收集相关的工具和命令参考

### 第3步: 内容创作
- 按照模板结构逐部分创作
- 确保语言工程化但不晦涩
- 保持内容的深度和实用性

### 第4步: 质量检查
- 使用质量检查清单逐项检查
- 确保案例的新鲜度和代表性
- 验证动手任务的可行性

### 第5步: 文档更新
- 使用 `scripts/update-learning-doc.py` 更新学习文档
- 确保格式正确
- 验证更新结果
